🎤 マイクがロングトーンで途切れるときの対処

ロングトーンで入力レベルがすぐゼロになる場合、多くはOS/ドライバやブラウザ側の自動ノイズ抑制/AGC/VADが原因です。以下を上から順にお試しください。

Windows（PC）
  • 設定 → システム → サウンド → 入力 → デバイスのプロパティ → 追加のデバイスプロパティ（コントロール パネル）で、ノイズ抑制/ビームフォーミング/自動ゲインを無効にする。
  • 録音デバイスの「詳細」で既定の形式（例: 48kHz 16bit）を固定。アプリの要求と揃えると安定する場合あり。
  • Zoom/Teams/Discord 等の常駐アプリは完全終了、あるいはアプリ内の「高度なオーディオ処理」をOFF。
macOS（Mac）
  • システム設定 → サウンド → 入力 → 「環境ノイズを低減」をOFF（OSバージョンで文言が異なる場合あり）。
  • 通話系アプリのバックグラウンド処理を停止、もしくはアプリ側のノイズ抑制/AGCをOFF。
iPhone / iPad（iOS/iPadOS）
  • コントロールセンター → マイクモードを「ワイドスペクトル」に変更（ノイズ抑制が弱まり持続音が途切れにくくなる場合あり）。
  • ブラウザは最新のSafari推奨。ホーム画面に追加（PWA）＋画面常時オンで安定することがあります。
Android
  • 通話強化や騒音抑制の端末設定をOFF（機種により名称が異なります）。
  • Chrome最新版を推奨。ホーム追加（PWA）＋画面常時オンで省電力の影響を受けにくくなる場合あり。
ブラウザのヒント
  • このアプリは可能な限り「生のマイク」を要求しています（AEC/NS/AGCをOFF指定）。ただし端末側で強制される場合は回避できません。
  • Chrome系では flags や実験的機能が影響することがあります。業務端末ではポリシーにより変更不可の場合あり。
外部マイク / オーディオIF
  • USBオーディオインターフェースや外部マイクは、内蔵マイクより自動処理が少ない傾向。ロングトーンの安定性が大きく向上します。
クイックチェック（困ったら）
  1. 通話アプリを完全終了
  2. OSのノイズ抑制/AGCをOFF
  3. 外部マイク/IFがあれば切替
  4. ブラウザ最新版で再試行（必要ならPWA＋画面ON）

注: これらは端末/OS/ブラウザの設計に依存します。アプリ側ではAEC/NS/AGCを無効化要求していますが、上流で強制される場合は端末設定の調整が必要です。

❓ このアプリの使い方
まずはこれだけ
  1. 上の「メロディ音声を選択」か「🎯 基礎練習モード」を選ぶ
  2. ▶ 再生 を押して、ガイドに合わせて声を出す（棒線が履歴）
  3. 止めたら「🧠 アドバイス」で改善ポイントを見る
音声を読み込む
  • 「メロディ音声を選択」から WAV / MP3 / M4A を選ぶ
  • 必要なら「伴奏音声を選択」も追加できる
マイクで練習
  • 「マイク許可」を押して、メーターが動くか確認
  • 小さすぎる音はゲート(dB)でカット。遅延は「🧭 アシスト」で合わせる
  • 長音が途切れる時は「📘 マイクの安定化ヘルプ」を見る
編集モード
  • 「✏️ 編集モード」ONで、再生線のノートが自動で選ばれる（黄色は単一選択）
  • 分割 / ±1 / ±12 / 範囲選択。保存→復帰でいつでも戻せる
採点とアドバイス
  • 棒線がガイドに近ければOK。許容は「採点」で調整できる
  • 停止後に「🧠 アドバイス」で傾向とヒントをチェック
見づらい・合わない時
  • 見づらい: 「表示」で縦範囲やガイド太さを調整（触らなくてもOK）
  • 精度を上げたい: 「解析」でレートやA4を調整（分からなければ既定でOK）
トラブル時のヒント
  • 音が拾えない: 入力レベル/ゲートを上げる。静かな場所で
  • 遅れて聞こえる: 「🧭 遅延補正アシスト」で合わせる
  • 長音が切れる: 「📘 マイクの安定化ヘルプ」を開く
